# Release Note: v0.2 (`beta0.2`)

## Release Summary
`v0.2` marks completion of **Stage II: Controlled Ecological Dynamics**.  
The engine now provides a deterministic ecological baseline with reproducible simulation behavior, regime validation, and reference-hash regression protection.

## Features Included
### Deterministic simulation core
- canonical state hashing via `sha256(get_state_bytes(engine))`
- deterministic agent iteration (`sorted(agent_ids)`)
- deterministic commit ordering (deaths -> births -> world regrowth)
- snapshot/restore continuation support
- isolated RNG streams (world, movement, reproduction, energy initialization)

### Ecological model
- 1D toroidal world topology
- fertility field + bounded resource regeneration
- movement energy cost and harvest-based energy recovery
- energy-gated stochastic reproduction
- death pathways: old-age, metabolic starvation, post-reproduction death
- capacity-aware population cap enforcement

### Experiment and validation infrastructure
- canonical regimes: `extinction`, `stable`, `saturated`
- batch runner with run-seed generation from master seed
- metrics + fingerprint analytics for validation
- regime-specific validation thresholds
- determinism suite with pinned reference hash baseline

## Determinism Guarantees
Under fixed `(seed, configuration, code version, runtime)`, the system guarantees:
- reproducible state trajectories
- canonical equality by state hash
- snapshot continuation equivalence
- RNG isolation between stochastic concerns
- seed sensitivity (different seeds diverge)

## Known Limitations
- spatial model is currently 1D only
- no explicit agent-agent interaction layer (competition is mostly implicit via update order/resource contention)
- no trait heterogeneity, inheritance, or mutation
- SeedSequence child-counter restoration is emulated (deterministic for current engine path, not full NumPy internal continuation)
- operational caveat: `python -m engine_build.main --mode validation --regime all` may fail in environments with plotting import issues before validation dispatch

## Next-Stage Scope (v0.3 / Stage III)
Primary focus:
1. explicit interaction and competition mechanics
2. stronger spatial structure (including optional 2D topology path)
3. new invariants and validation extensions while preserving determinism guarantees
