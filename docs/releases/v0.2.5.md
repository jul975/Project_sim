# Release Note: v0.2.5

## Pre-Stage III Freeze Baseline

`v0.2.5` marks the documented pre-Stage III freeze checkpoint for the current repository state on March 23, 2026.

This is not the Stage III release. It is the cleaned-up freeze artifact that closes out the current Stage II line and defines the baseline that Stage III work should build on.

Important versioning note:

- package metadata in `pyproject.toml` is currently `0.3.0a0`
- this release note uses `v0.2.5` as the project-level freeze label for the pre-Stage III baseline

## Release Summary

This freeze baseline now includes:

- deterministic 2D toroidal world topology
- phase-structured engine flow:
  - movement
  - interaction
  - biology
  - commit
- canonical state hashing and snapshot continuation
- compact deterministic agent identity and RNG partitioning
- batch experimentation with post-hoc regime classification
- CLI lanes for `experiment`, `verify`, `validate`, and `menu`
- refreshed canonical and project-status documentation aligned to the live code

## What Changed Since `v0.2`

Compared with the original `v0.2` Stage II note, the current freeze baseline reflects a materially more mature runtime.

Historical reference:

- [v0.2.md](./v0.2.md)

### 1. 2D topology is now the live engine model

The world, movement logic, snapshots, hashing, and exported world views all operate on 2D `(x, y)` positions.

### 2. The birth path was refactored

The heavy earlier newborn construction path was replaced by:

- compact deterministic identity words
- direct newborn positioning at the parent location
- cleaner separation of identity, RNG, and state initialization

### 3. The CLI surface was simplified

The public command lanes are now:

- `experiment`
- `verify`
- `validate`
- `menu`

The older fertility-lane documentation is no longer authoritative for the current tree.

### 4. Verification and validation are both first-class

The repository now has:

- verification suites for determinism, snapshots, invariants, and RNG isolation
- validation suites for regime contracts and regime separation
- CLI dispatch into both testing lanes

### 5. Analytics and observability are stronger

The current baseline includes:

- run and batch fingerprints
- regime summaries and post-hoc classification
- optional performance profiling
- optional world-frame sampling and world-frame analytics

## Current Regime Set

The live named regimes are:

- `saturated`
- `collapse`
- `extinction`

## Test Status

Checked in the project virtual environment on March 23, 2026:

- full pytest run: `31 passed`

That is the key freeze-state difference from the earlier March project snapshot: the repository is no longer carrying a known failing validation test at the status-doc level.

## Performance Status

The earlier abundant-run slowdown is no longer blocking development.

Documented reference benchmark improvement:

- legacy abundant reference: `1682.87s`
- refactored abundant reference: `19.47s`

Interpretation:

- the performance crisis is resolved
- births remain the main hotspot in growth-heavy runs
- performance is still a future optimization topic, but not a freeze blocker

## Known Limitations

This is a freeze baseline, not a claim that Stage III is already done.

Current known limits:

- Stage III crowding, collision, and local-competition rules are not implemented yet
- `tail_fraction` exists on the experiment request but is not yet passed into `AnalysisConfig` by `run_experiment_mode()`
- snapshot restore currently forces `collect_world_view = False`
- `landscape_spec.contrast` and `landscape_spec.floor` are not yet applied in fertility generation
- validation contracts are still selective rather than exhaustive across all six named regimes

## Recommended Usage

Baseline experiment:

```bash
python -m engine_build.main experiment --regime stable
```

Repository-level checks:

```bash
python -m engine_build.main verify --suite all
python -m engine_build.main validate --suite all
```
